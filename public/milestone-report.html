Stage Summary (Review of Previous Stage)

The Project Manager should report the progress achieved during this stage, including any problems or unexpected issues and their impact on the project, any costs incurred or lessons learned, and whether the project has remained on schedule.

During the current development and testing stage, the SmartWaste Route system has achieved significant progress across all planned deliverables. The following key milestones were completed:

  • Admin Management System: Complete CRUD operations for barangays, zones, residents, collectors, and trucks with premium UI design, pagination, search, and filters.
  • Route Optimization: Google Directions API integration with batch processing for 25+ waypoints, automatic collector-truck assignment, and date validation.
  • Map Integration: Interactive polyline-based barangay boundary visualization with click-to-zoom, glow effects, and real-time truck tracking for residents.
  • Collection Reports: Full analytics dashboard with area charts, donut charts, bar charts, period filtering, and export-to-print functionality.
  • Resident Features: GPS-based auto-registration with geofencing (Tuy bounds check), automatic zone assignment via nearest-centroid algorithm, missed pickup reporting, and responsive mobile/desktop layouts.
  • Dark Mode: Complete dark mode support across all admin and resident interfaces with emerald-tinted color scheme.
  • Schedule Management: Monthly calendar view with route visualization, sidebar details, and status tracking.

Problems encountered during this stage included OpenStreetMap Nominatim API rate limiting (resolved with retry mechanism and centroid-based detection), Google Directions API waypoint limits (resolved with batch splitting), and household count discrepancies for multi-barangay zones (resolved by querying through pivot tables). All issues were identified during testing and resolved before milestone completion.

Has the project remained on schedule/on budget/within scope?

The Echo Haulers project remained on schedule, within budget, and within scope during the development and testing stage. Planned tasks were completed on time, project costs stayed within the allocated budget, and all core system features were delivered as outlined in the project plan. Minor refinements were made based on testing feedback, including:

  • UI redesign across all admin pages for premium look and consistency
  • Addition of truck management module (not in original scope but required for route assignment)
  • Geolocation-based registration restriction to Tuy, Batangas municipality

These additions were accommodated within the existing timeline through efficient development practices and did not significantly affect the project schedule or budget.

Business Case Review

The SmartWaste project was developed to improve waste management in Tuy, Batangas by addressing issues in manual reporting, delayed response, and lack of real-time monitoring. It aims to replace inefficient processes with a centralized system that improves accuracy, speed, and coordination.

Options Considered:

  1. Maintaining the current manual system
  2. Making partial improvements to existing processes
  3. Fully developing the SmartWaste Route system

Option 3 was chosen because it offers the most effective long-term solution with automated scheduling, GPS-based route optimization, real-time tracking, and data-driven reporting.

Expected Benefits: Faster waste reporting, better monitoring, reduced manual work, fewer errors, improved decision-making through real-time data, optimized collection routes reducing fuel costs and time.

Risks Identified: Technical challenges (API rate limits, waypoint limits), user resistance, data errors, budget limits, system downtime, and security concerns. All identified risks have been managed through mitigation strategies.

Status: The project is proceeding according to plan. All core features have been implemented and tested successfully.
