1.0 INTRODUCTION

1.1 Purpose

The Test Report provides a detailed summary of the testing activities conducted for the SmartWaste Route: Waste Collection Scheduling & Route Optimization System. It documents the test cases performed, expected and actual results, identified defects, and the corrective actions taken during the testing process. This report is intended to evaluate the system's functionality, reliability, performance, and usability to ensure that all major features operate according to the specified requirements. The results of this report will determine whether the system is ready for deployment and practical use in improving waste collection operations in Tuy, Batangas.

3.0 TEST ASSESSMENT

The testing for the SmartWaste system adequately covered its main features, including scheduling, route optimization, tracking, reporting, and dashboard functions. Results showed that the system performs reliably and meets its core requirements. The geolocation-based registration and automatic zone assignment were tested with GPS coordinates within Tuy, Batangas and functioned correctly. The route optimization using Google Directions API was validated with up to 50+ waypoints through the batch processing implementation. Overall, the test was sufficient and indicates that the system is ready for deployment with minor refinements.

5.0 VARIANCES

The following variances were observed between planned and actual testing:

  • Geocoding API Rate Limiting: OpenStreetMap Nominatim returned 429 errors during rapid testing. This was resolved by implementing a retry mechanism with exponential backoff and a graceful fallback response.
  • Google Directions API Waypoint Limit: The planned single-request route optimization failed with zones containing 25+ households. This was resolved by implementing batch processing that splits waypoints into groups of 25 with overlap stitching.
  • Household Count Discrepancy: Zones with multiple barangays initially showed 0 households because the query only checked zone_id. This was fixed by also querying through the barangay pivot table.

7.0 RECOMMENDATIONS

  1. Load Testing: Conduct stress testing with concurrent users to evaluate system performance under heavy traffic, particularly for real-time truck tracking and notification polling.
  2. Field Testing: Deploy in a controlled pilot area within Tuy to test GPS accuracy, mobile responsiveness, and collector route navigation under real-world conditions.
  3. Network Resilience: Test system behavior in areas with poor network connectivity, especially for the collector mobile interface during route collection.
  4. Data Backup: Implement automated database backup procedures before production deployment.
  5. User Training: Provide training sessions for administrators and collectors before full deployment to ensure smooth adoption.
